Best Practices Analyzer Core Library
The Run Element

This element is generated on output and it identifies a particular run that generated the data file and contains specific parameters that were specified in that run.  A data file may be reprocessed multiple times, for instance to reanalyze data that was previously collected and analyzed with an earlier set of rules.  There will be one Run element for each of these runs.

Example Configuration Fragment

In an output data file:

<Run
   ConfigName="ExBPA Master Input File"
   ConfigVersion="1.5.6.1"
   AppVersion="1.0.7408.1"
   StartTime="09/30/2004 23:20:39"
   Timeout="240"
   Threads="25"
    Operations="collect,analyze"
   EndTime="09/30/2004 23:25:48"
   Completed="OK"
   ObjectsTracked="9"
>

Parent Elements

Element Description
Configuration This element contains some general meta-information about the input configuration or the output data.

Child Elements

Element Description
Log This output element contains the text of a single logged message.

Attributes

Attribute Description
AppVersion This is the version of the application that ran the scan.
CompletedThis indicates the results of the scan done.
ConfigNameThis is name of the configuration file at the time the run was started.
ConfigVersionThis is the version of the configuration file at the time the run was started.
EndTime This indicates the time the run completed.
ObjectsTrackedThis indicates how many objects were marked for tracking in the latest run.
Operations This is a list of operations that were selected for the run.
StartTimeThis indicates the time the run was begun.
Threads This is the maximum number of threads specified for the run.
Timeout This is the default timeout specified for the run.
