The SDMMultiplicity Attribute

SDM (Systems Definition Model) is a strategic initiative to create an all encompassing language and processing engine for modeling applications.  This will be a key part of System Center and will be used for configuration management, system monitoring, performance analysis, security verification, and other similar functions.  BPA is considered an interim solution until this broad system is delivered (this will likely be several years down the road).  To avoid having to redo all the knowledge that will be encoded in BPA configurations during the interim, it is desirable to be able automatically convert BPA configurations to SDM documents.  While this can be done without additional hints, the SDM document that gets created will likely not be optimal for SDM, although it will be functional. 

To enable better conversions, a set of hinting attributes have been created.  These attributes are optional and not recognized by the BPA engine, but it is recommended that they be applied so that the future conversion to SDM goes as smoothly as possible. 

In BPA, both Settings and Objects can have zero or more values (in the case of Settings) or instances (in the case of Objects).  In SDM, you can specify a Multiplicity attribute on the equivalent elements.  This attribute can be used to define how many values or instances are expected.  By default, this will be set to "*", which indicates zero or more are allowed.  The SDMMultiplicity attribute can be used to override this.

Example Configuration Fragment

<Object
   Type="WMI"
   Key2="root\directory\ldap"
   Key3="rootDSE"
   Key5="*"
   SDMMultiplicity="0..1"
>

Parent Element

Element Description
Object This attribute is specified on input Object elements.
Setting This attribute is specified on input Setting elements.

Possible Values

Element Description
<lower..upper> This provides the lower bound and upper bound of valid number of instances or values.
<upper> This provides the upper bound of valid number instances or values (the lower bound is 0).
