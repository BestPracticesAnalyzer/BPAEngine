Best Practices Analyzer Core Library
Function Processors

Function processors are used to a define special functions that can be referenced by Rule elements.  These functions can simplify or enable complex queries without having to encode complex XPath logic.  They are defined using ExtFunction elements in the configuration.

A number of built-in function processors are supplied in the SDK (these do not actually use the extension logic so they do not need to be defined in the configuration, but the concept is the same):

upper-case(p) This converts the parameter to all upper-case characters.
lower-case(p) This converts the parameter to all lower-case characters.
hex(p) This interprets the parameter as a hex string and returns the corresponding number.
matches(p1, p2) This returns true if string p1 satisfies regular expression p2, otherwise it returns false.
replace(p1, p2, p3) This replaces all occurances of the character pattern specified by the regular expression in p2 in string p1 with string p3.
join(p1, p2) This joins all members of the p2 node set into a string using p1 as a separator.
slice(p1,p2,p3) This returns the p3th element in the string p1 where p2 is the element separator.
bitwise-and(p1, p2) This returns the bitwise logical and of the the two parameters.
bitwise-or(p1, p2) This returns the bitwise logical or of the the two parameters.
if(p1, p2, p3) This returns p2 if p1 when evaluated as a boolean is true, otherwise it returns p3.
date-difference(p1, p2) This returns the time difference for two dates.
date-add(p1, p2) This returns the date specified by adding the timespan p2 to date p1.
date-parse(p1, p2) This translates a date p1 in format p2 to the standard "MM/dd/yyyy HH:mm:ss" format.
get-days(p) This returns the number of days in the timespan p.
get-seconds(p) This returns the number of seconds in the timespan p.
sdcheck(...) This return true if a security desciptors DACL contains the specified ACE.  See the implementation for more details.

These additional function processors use the extension logic and so need to be configured to be accessible:

list-average This will return the average of all the values in a list.
list-max This will return the maximum of all the values in a list.
list-median This will return the median of all the values in a l;ist
list-min This will return the minimum of all the values in a list.
list-percentile This will return the value at a given percentile of all the values in a list.
list-sigma This will return the sigma result of all the values in a list.
