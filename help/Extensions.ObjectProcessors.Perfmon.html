Best Practices Analyzer Core Library
The Perfmon Object Processor

This is used to do simple queries of performance counters.  For some purposes this is useful and more straightforward to use than Pdh, but it is a point in time snapshot of a counter value, so it is somewhat limited.

One Instance element per Perfmon instance will be created and the Setting elements will be processed separate for each one.  The Value elements will contain the Perfmon counter values retrieved in each case.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="Perfmon"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.PerfmonObjectProcessor"
/>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1

This is the name of the server to access.
Key2 This is the name of the Perfmon category to access.
Key3 If this is blank or not given and the category has only one instance, then this will match that one instance.  Otherwise, it will match against all instances of the category that match the regular expression. This can also be set to a specific instance name in which case it will only match against that instance.
Type This must be set to "Cache".
