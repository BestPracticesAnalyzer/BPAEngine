Best Practices Analyzer Core Library
The Object Element

This element identifies an entity to be processed by an object processor.  This might represent something like an AD query (such as give me all Exchange servers in the configuration container), or it may be a test type (such as a protocol test).  The specific definition is up to the object processor. 

Objects are nested under each other and form an object hierarchy.  The object hierarchy forms the basis of the configuration structure.  When an Object is processed by an object processor, that processor will return zero or more instances of that object, and for each instance it will fill in the values of the settings defined for the object.  The setting values retrieved can be further used in a variable which is called a Substitution.  Through judicious use of nested objects and substitutions, the configuration can be made to be very flexible and powerful.

Example Configuration Fragment

In the configuration file:

<Object
   Type="Directory"
   Key2="cn=Exchange Enterprise Servers,cn=Users,%DOMAINDN%"
>

In an output data file:

<Object
   Type="Directory"
   Key2="cn=Exchange Enterprise Servers,cn=Users,%DOMAINDN%"
   ProcessTime="10.0376"
>
     <Instance
          Type="Directory"
          Key2="cn=Exchange Enterprise Servers,cn=Users,DC=contop,DC=com"
          Name="Exchange Enterprise Servers"
       >
       ...

Parent Elements

Element Description
root This element can be just below the XML root node.
Instance On output, Object elements are nested under Instance elements of their parent Object.
Object On input, Object elements can only be nested under another Object element, or at the root of the document.
Reference This element refers to a Type element to be included in that position.

Child Elements

Element Description
Instance On output, Instance elements are created under Object elements (one per matching instance).
Setting On input, Setting elements are nested directly under Object elements.  These elements contain data about a particular property to be retrieved.

Attributes

Attribute Description
Async This indicates how many threads should be spun up to process the instances of this object.
Class This provides a display hint to the GUI for showing the object.
Count Identifies a property to count over.
CustomId This is an identifier that can be referenced by a Customize element to override its properties.  This is only meaningful if the Object is nested under a Type.  This defaults to the Name of the Object.
Display This property is used as a display hint for the object.
Filter This is an internal attribute, only set on output, that acts as a filter on the Name of instances (based on Scope).
GroupingClass This indicates that the object should be used to group issues for display purposes.
Key1 - Key5 These values can be given any meaning by the object processor associated with this type of object.
LogText This is informational text to log when this object is dispatched for processing.
MaxFailures This can be used to limit the number of failures that will be tolerated on any queries underneath this object.
MaxTime This can be used to limit the total time that will be tolerated to process all the queries underneath this object.
MaxTimeouts This can be used to limit the total timeout failures that will be tolerated to process all the queries underneath this object.
Name This provides information on the name of the object.
ProcessTime This output only attribute contains the amount of time it took to process the object.
Range This identifies the maximum number of instances to include in the output.
Restrict This is an internal attribute, only set on output, denoting an whether an element should be ignored durring processing due to restrictions (e.g., Category, Scope, or Level).
<restriction> This is a custom-defined restriction attribute.
SDMElement This provides a hint to be used during conversion of a BPA Object element to an SDM element.
SDMMultiplicity This provides a hint to be used during conversion of a BPA Object element to an SDM element.
SDMName This provides a hint to be used during conversion of a BPA Object element to an SDM element.
SecurityContext This specifies the security context to use when processing this object.
Sort This determines how to order the instances in the output.
Timeout This is the timeout to be used for this object.
TrackProgress This indicates that the core engine should track the progress of this object and provide status updates as processing occurs.
Type This is the type of the object which is used to determine which object processor will process it.
