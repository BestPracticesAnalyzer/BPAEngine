Best Practices Analyzer Core Library
Common Models

The BPA SDK provides a set of common types that can be referenced by other configurations.  These are defined in BPA.CommonTypes.xml.  These are useful for creating standard definitions that can be used by multiple applications.  These standard definitions will also come into play in the future when BPA configurations need to be converted to SDM configurations (the Systems Definition Model is a strategic effort being driven out of System Center that is being designed to handle best practice definitions and much more).  See the BPA to SDM Conversion Document for details. 

Note that the elements that make the common model functional are Type, Reference, and Customize, and these only processed when the Type Configuration Preprocessor is configured.  See the built-in Configuration Preprocessor extensions for details.

Here are the common types that are supplied with the SDK (these are more completely described in comments in the configuration file):

BIOS This gathers information on the BIOS of a computer.
CacheMemory This gathers information on all memory caches configured on a computer.
Computer This pulls in hardware information on a computer (it references many of the other Types).
Controllers This gathers information on all controllers configuration on a computer.
File This gathers information on a single file.
LogicalDisks This gathers information on all logical disks configured on a computer.
OS This gathers information on the OS of a computer.  It also references the OSUpdates Type.
OSUpdates This gathers information on the OS updates installed for a specific version of the OS.
PhysicalDisks This gathers information on all physical disks installed on a computer.
Processors This gathers information on all processors installed on a computer.
Server  
Volumes This gathers information on all volumes configured on a computer.
