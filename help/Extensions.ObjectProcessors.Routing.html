Best Practices Analyzer Core Library
The Routing Object Processor

This uses ICMP to determine response times and route hops between machines.  This will create a single instance in the output file, with the values for the setting containing the routing information.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   Name="Routing"
   Assembly="BPA.NetworkCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.RoutingObjectProcessor"
/>

An example object:

<Object Type="Routing" Name="Routing" Key1="%SRVNAME%" Key2="30" Key4="60000">
   <Setting Key1="RouteMap" />
</Object>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1 This is the name of the server to access
Key2 This is the max hop count to allow before giving up.
Key3 This is no longer used.
Key4 This is the max response time to allow before giving up (in milliseconds).
Type This must be set to "WMI".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1 This is the command to execute.  Currently, only "RouteMap" is defined.  The values returned for this setting will be a sequence of node name and response time values, in order of traversal.
