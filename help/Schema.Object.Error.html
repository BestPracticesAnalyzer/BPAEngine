Best Practices Analyzer Core Library
The Error Attribute

This attribute indicates the level of importance assigned to a particular issue.    This can be used for reporting purposes.  Note that the possible values listed for this attribute are recognized by the Common UI layer, but not by the engine.  If you are not using the common UI, then you can set these values to anything you want.  One caveat: "Error", "Warning" and "Info" are also recognized by the Events issue processor (see the Issue Processors topic) and determine what level is assigned to the event.

Example Configuration Fragment

<Rule
   Name="fCTFoundAdminGroup"
   Query="$."
   Error="None"
   Text="Found Admin Group {1} in the directory." P1="$."
/>

Parent Element

Element Description
Message This is copied from the rule that created it.
Rule This is defined on input and indicates the error level to apply to this rule.

Possible Values

Value Description
"Error" This means the issue is of the highest importance and very likely represents a real problem that needs to be corrected.
"Warning" This is slightly less severe and indicates something that may be wrong in certain circumstances, but not all.
"BestPractice" This indicates a best practice to get the most optimal or reliable use out of a deployment.
"Baseline" This is a special error type indicating a comparison of a property against a preset value has failed.
"Time" This indicates a property that has been recently changed.
"NonDefault" This indicates a property that, while not necessarily wrong, has been altered from its default value.
"None" This indicates a purely informational message.
<not present> This indicates no issues have been fired (on output), and on input Rule elements it indicates the rule is for internal use only.
