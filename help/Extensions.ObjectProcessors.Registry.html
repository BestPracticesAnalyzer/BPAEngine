Best Practices Analyzer Core Library
The Registry Object Processor

This will retrieve data from the Windows registry.

If a regular expression is given in Key4 or a scope of "OneLevel" or "Subtree" is given in Key5 on the Object, then this will create zero or more Instance elements.  If neither is the case, it will create zero or one Instance elements.  The Name attribute for each instance will be set to the key path of the associated registry key.  Each Setting will be processed separately per Instance and will have zero or more Value elements added to them based on the contents of the corresponding registry value.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="Registry"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.RegistryObjectProcessor"
/>

An example object:

<Object
   Type="Registry"
   Key1=""
   Key3="Software\Microsoft\NET Framework Setup"
   Key4=".*"
   Key5="Subtree"
>
     <Setting Key1="Install" />

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1 This is the name of the server to access. If missing or blank, the local machine is used.
Key2 This is the registry hive to access.  This will default to HKEY_LOCAL_MACHINE. 
Key3 This is the registry key to access.
Key4 This is a filter string for the registry lookup. If this is blank or not given, this object will match against the exact registry key given.  Otherwise, it will match against all the child keys of the given registry key that match the regular expression in the scope of the search.
Key5

This is a search scope to use.  It can have one of the following values:

  • "Base" means just the exact object defined by the given key.  This is the default if no filter is specified.
  • "OneLevel" means to just match against the immediate child objects under this one.  This is the default if a filter is specified.
  • "Subtree" means to match against the whole tree under this object.
Type This must be set to "Registry".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1 This is the name of the registry value to retrieve.  "(Default)" is converted to "" to match RegEdit's semantics.
