Things to Consider Before Shipping a BPA to Customers

Overview

This topic points out a few things that need to be done before a BPA is ready to be shipped out.

Preparation Steps

The following is a list of things that will need to be taken care of or should otherwise be considered before embarking on shipping a BPA tool:

  • Create an MSI for the install.
  • Create a help file (hopefully your UE team takes care of this).
  • Create all the detailed articles and make sure the links are live (this would absolutely be done by the UE team).
  • If you are going to localize to any languages other than English, take care of that.
  • Go through the Checkpoint Express process to get your tool blessed for release.  This takes some time and includes running several different scans on any custom code you've got as well as getting a EULA, privacy statement, security review, etc.
  • Establish an external feedback alias and, if possible, a newsgroup to receive questions and post responses to.Establish a channel with your support engineers to receive feedback and new rule suggestions.
  • Set up your web site for automatic update handling (if you are going to be supporting this feature).
  • Advertise the existence of the tool – you may want post a blog entry somewhere about it, or perhaps your marketing team is willing to do some work.
  • Testing – bug bash/dedicated test/dev test.
  • Globalization – make sure your BPA works well in let’s say German OS with Japanese product.
  • OS platform – make sure your BPA works on win2k (there may be some WMI issues here).
