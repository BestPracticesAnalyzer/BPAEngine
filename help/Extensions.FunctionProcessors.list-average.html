Best Practices Analyzer Core Library
The list-average Function Processor

This function processor can be used to get the average value of the integer nodes in a comma-separated list.  This was designed specifically for use with the Pdh object processor to get an average value across performance samples, but it can be applied to any list of integers.

Example Configuration Fragment

The FunctionProcessor definition:

<ExtFunction
   Name="list-average"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.PdhObjectProcessor"
   Function="Average"
   Returns="Number"
   Parameters="String"
/>

An example usage:

<Setting
   Key1="SMTP Server"
   Key2="Cat: Categorizations in progress"
   Key3="_Total"
>
   <Rule
      Name="fAvgCatInProgAt1000"
      Query="list-average($.) &gt;800 and list-average($.) &lt;1200"
      Error="Error"
      Sev="2"
      Title="Messages are submitted faster than categorizer can process"
      Text="The 'Cat: Categorizations in progress' counter on server {2} stays steady around 1000."
       S2="%varExName%"
   />
</Setting>

Specific ExtFunction Attributes (see ExtFunction for a full attribute list)

Function

This must be set to "Average".
Returns This must be set to "Number".  This returns the average value.
Parameters This must be set to "String".  This takes a comma-separated list of integers.
