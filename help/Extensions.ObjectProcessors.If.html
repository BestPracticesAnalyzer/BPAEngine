Best Practices Analyzer Core Library
The If Object Processor

This object can be used to evaluate simple boolean conditions. 

This will create one Instance element if the condition is true, zero if false. The Name attribute for the instance is taken from the object.  Setting elements are not required but if present will be copied to the data document with an empty value.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="If"
   Assembly="BPA.Common.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Common.IfObjectProcessor"
/>

An example object definition:

<Object
   Type="If"
   Name="Test"
   Key1="'%SRVNAME'='myserver'"
   >

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Name This is just used for informational purposes.
Type This must be set to "If".
Key1 This is the boolean condition to test. It can contain an valid xpath or extension function, but must refer only to substitutions, constants, and data on this object node.

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1 This attribute is evaluated as xpath as stringified into a single value. This is typically useful for capturing substitutions that can't be expressed simply as a format.
