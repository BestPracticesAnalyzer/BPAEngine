Best Practices Analyzer Core Library
The Schema Definition

This section described all the elements and attributes that comprise the configuration that the BPA engine consumes.

Top Level Elements

Element Description
Configuration This element contains some general meta-information about the input configuration or the output data.
Object This element contains information about a single item to be processed.
Reference This element refers to a Type element to be included in that position.
Type This element defines a reusable configuration fragment.

All Elements

Element Description
ConfigPreprocessor This element identifies a managed class that will be used to preprocess the configuration. 
Configuration This element contains some general meta-information about the input configuration or the output data.
Customize This element can be used to alter specific portions of referenced types.
DataPostprocessor This element identifies a managed class that will be used to postprocess the data document after collection and analysis has been completed.
ExtFormat This element maps an Setting.Format to a managed class.
ExtFunction This element maps an custom rule function to a managed class.
IssueProcessor This element identifies a managed class that will be invoked when a rule fires.
Instance This output element contains an instance of an input Object.
Log This output element contains the text of a single logged message.
Message This output element is generated when a rule fires.
Object This element contains information about a single item to be processed.
ObjectProcessor This element maps an Object.Type to a managed class.
Option This element identifies an option of a restriction.
Pivot This element allows a indirect referencing between rules.
Reference This element refers to a Type element to be included in that position.
RestrictionType This element identifies a restriction to apply to the configuration.
Result This output element contains the results of a rule that got processed.
Rule This element contains information about a rule to execute on the output data.
Run This output element contains general information about a particular run.
Setting This element identifies a property to be retrieved from an Object.
Substitution This element contains a parameter value that can be references elsewhere in the configuration.
Template This element identifies a rule template that can be applied to other rules so similar rules don't need to be defined separately.
Type This element defines a reusable configuration fragment.
Value This output element contains the string contents of a single value of a Setting.

These elements are all arranged in a hierarchical fashion to form the BPA configuration or the output data file (some elements appear in both the input configuration and the output data files, and some just appear in one or the other).  Each element description defines the purpose of the element, all the valid attributes, and what are valid child elements are.
