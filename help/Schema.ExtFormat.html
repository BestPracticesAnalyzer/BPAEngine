Best Practices Analyzer Core Library
The ExtFormat Element

This element identifies the functions to be loaded at run time which are invoked when specified by a particular setting.  These are used to define custom formats for converting or displaying data.  The Assembly/Class/Convert triplet defines the Convert function, which is used during data collection to reduce data to the format that will be persisted.  The Assembly/Class/Display triplet defines the Display function, which is used during data rendering to expand data to a more human-readable format.  If the two functions are in different classes, simply specify two ExtFormat elements with the same Name.

See Format Processors for a complete description of the extensions that are included in the SDK.

Example Configuration Fragment

In the configuration file:

<ExtFormat
   Name="SecurityDescriptor"
   Assembly="BPA.Common.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Common.NTSD"
   Display="Display"
/>
 

Parent Elements

Element Description
Configuration ExtFormat elements only exist in the input configuration.  They are not echoed to the output data file.

Child Elements

Element Description
none This element has no children.

Attributes

Attribute Description
Name This is the name used in a query to invoke the function.
Assembly This should point to a DLL that contains a managed assembly which implements the function.
Class This is the name of the managed class that implements the function.
Display This is the name of the managed function that implements the convert function.
Convert This is the name of the managed function that implements the display function.
