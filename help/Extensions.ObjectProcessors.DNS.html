Best Practices Analyzer Core Library
The DNS Object Processor

This retrieves data through Win32 DnsQuery call.

For each DNS query instance found that matches the given query, an Instance element will be created and all the elements under the original Object will be duplicated in each Instance.  Each Setting will be processed separately per Instance and will have zero or more Value elements added to them based on the contents of the corresponding DNS property.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   Name="DNS"
   Assembly="BPA.NetworkCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.DNSObjectProcessor"
/>

An example object:

<Object
   Type="DNS"
   Name="DNS"
   Key1="%DNSSRVIPS%"
   Key2="%SRVNAME%"
   Key3="%Options%"
>
   <Setting Key1="A" />
   <Setting Key1="PTR" Key2="%EXIPADDRESS%"/>
   <Setting Key1="MX" />
   <Setting Key1="SRV" Key2="_LDAP._TCP."/>
</Object>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1 This is the DNS server IP address.
Key2 This is the name of the server to look for.
Key3 This is the query options, such as UseTCPOnly. All options are seperated by comma.
Type This must be set to "DNS".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1 This can be one of the following:
  • "A" returns the fully qualified domain name (FQDN) of the given server.
  • "PTR" returns the FQDN of the specific NIC given in Key2.
  • "MX" returns the primary domain name of the given server.
  • "SRV" returns the primary domain name of the given server using the protocol specified in Key2.
Key2 If Key1 is "PTR", this should be the IP address of the NIC you are looking for.  If Key1 is "SRV", this should be the protocol to use.
