Best Practices Analyzer Core Library
The ObjectProcessor Element

This element identifies an object processor module to be loaded at run time and maps that module to an object type.  These are used to process objects in the configuration during the data collection phase.

Several object processors are delivered with the BPA library.  See Object Processors for a complete description of these.

Example Configuration Fragment

In the configuration file:

<ObjectProcessor
   Name="Registry"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.RegistryObjectProcessor"
/>
 

Parent Elements

Element Description
Configuration ObjectProcessor elements only exist in the input configuration.  They are not echoed to the output data file.

Child Elements

Element Description
none This element has no children.

Attributes

Attribute Description
Assembly This should point to a DLL that contains a C# assembly which has one or more object processors defined in it.
Class This is the name of a class that has implemented the ObjectProcessor logic.
Name This identifies the particular data postprocessor. It also serves to determine the sequence the postprocessors will be executed in (alphabetically by this value).
ObjectType This attribute means the same as Name, but is discontinued.
