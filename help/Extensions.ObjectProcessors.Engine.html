Best Practices Analyzer Core Library
The Engine Object Processor

This will retrieve meta-data about the run itself and the execution environment.  This will always create a single Instance element.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="Engine"
   Assembly="BPA.Common.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Common.EngineObjectProcessor"
/>

An example object:

<Object Type="Engine">
     <Setting Key1="EnvironmentVariable"  Key2="PATH"/>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Type This must be set to "Engine".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1 This is the name of the property to retrieve.  This can be:
  • AppVersion.  This will return the w.x.y.z version of the application.
  • ConfigVersion.  This will return the w.x.y.z version of the main configuration file.
  • DataFileName.  This will return the name of the data file that will be created for this run.
  • ExecutionDirectory.  This will return the path to the executable wrapper currently running the scan.
  • ApplicationName.  This will return the name of the executable wrapper currently running the scan.
  • StartTime.  This will return the time the scan was started.
  • ExecutionCulture.  This will return the locale being used for the current scan.
  • EnvironmentVariable.  This will return the value of the environment variable that is specified in Key2.
Key2 If Key1 is set to "EnvironmentVariable", then this should be set to the name of the environment variable to be retrieved.
