Best Practices Analyzer Core Library
The CheckTokenMemberShip Object Processor

This will create a single instance if the given SID is a member of the currently logged on token of the thread processing the object.  If it is not, no instances will be returned.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   Name="CheckTokenMembership"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.CheckTokenMembershipObjectProcessor" />

An example object:

<Object
   Type="CheckTokenMembership"
   Key1="S-1-5-32-544"
   Name="LocalAdmin"
>
   <Rule
      Name="fNotLocalAdmin"
      Query="count(../Instance[@Name='LocalAdmin'])=0"
      Error="Error"
      Text="The logged-on user ({1}) is not a member of the local Administrators group."
      S1="%CURRENTLOGON%"
      ErrorID="1008"
   />
</Object>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1 This is a stringified version of a SID to look for.
Type This must be set to "CheckTokenMembership".

Specific Setting Attributes (see Setting for a full attribute list)

    None - all settings will just be copied over to the data file with a single value containing an empty string.
