Object Processors

Object processors are used to access data from a data source of some kind.  They are also referred to as data source collectors or just collectors in some cases.  Object processors are defined at the start of a configuration document using ObjectProcessor elements, and then Object elements may be defined throughout the document that reference that processor.  The definition of an object processor is quite flexible.  In addition to the classic usage, which is to retrieve data from store of some kind (i.e. the registry, Active Directory, etc), they can take pretty much any action and can return various types of data.  For instance, the supplied Protocol object processor identifies a port number and commands to be sent rather than a query and properties to retrieve, and instance of return the values of properties it returns the responses to the commands.  Meta-type logic can also be implemented, such as in the supplied Group object processor, which provides a place holder that can be used to organize the configuration.

A number of different object processors have been created and can be used by all the product teams.  Together, these should cover most product needs.  The available common object processors are:

Cache This is a special processor that will cache data gathered during a run.
Certificate This extracts data from certificates via several mechanisms.
CheckTokenMembership This determines if a SID is a member of the logged on token.
CommandLine This executes a command line application.
Directory This enables access to Active Directory data.
DNS This enables DNS queries.
DsGet Can be used to get the site name for a server using the DsGetSiteName API (from netapi32.dll).
Engine This retrieves information about the run itself.
Enumerator This is a special object processor that creates a specified number of instances.
ExpandGroup This gets all the security groups a user is a member of.
File This enables access to flat files, XML files, and INI files.
GetUserNameEx This retrieves the currently logged on user name.
Group This is a special processor that allows a configuration to be better organized.
If This is a special processor that creates an instance if a condition evaluates to true.
LookupAccount This converts a SID to a user name.
Metabase This enables access to IIS metabase data.
Pdh This allows for basic Perfmon sampling.
PdhCollect This enables collecting perfmon data to a blg file.
PdhInfo This breaks up a large blg file into smaller chunks.
Perfmon This allows for basic single time access to Perfmon counters.
Port This is used to measure whether we can connect to a remote port or not.
Protocol This is used to measure whether the internet protocols are working properly or not.
Registry This enables access to information from the registry.
Resolve This object can be used to automatically choose between resolution methods (WINS or DNS) depending on which one succeeds.
Routing This retrieves routing information (such as hop count) to a given machine.
SQL This enables access to SQL databases.
WMI This enables access to Windows Management Interface classes.
