The MaxFailures Attribute

This indicates how many failures (meaning exceptions) will be allowed before the engine gives up processing this object and all objects underneath it.  This can be used to keep an inaccessible machine or interface from holding up a run too much.  Other similar object attributes are MaxTimeouts and MaxTime and another attribute that can be used for this is the Retrieve attribute on a Setting.

Example Configuration Fragment

<Object
   Type="Directory"
   Key2="%DN%"
   Key3="(&amp;(objectCategory=msExchExchangeServer)%SERVER%)"
   Class="Server"
   GroupingClass="True"
   Async="%65"
   TrackProgress="TrackProgress"
   MaxTime="ALL,1"
   MaxFailures="WMI,1"
   MaxTimeouts="ALL,1"
>

Parent Element

Element Description
Object This attribute is specified on input Object elements.

Possible Values

Value Description
"<object>,<count>[;...]"

This is a semi-colon separated list of comma separated pairs that each define a single object type and a max failure count.  The object type can be any type that has been defined by an object processor in the configuration file.  It can also be "ALL" to indicate that this applies to all types of objects.  When there have been <count> failures to the specific object type, any further objects of that type defined in the same hierarchy will be skipped.
