The ConfigVersion Attribute

This is the current version of the configuration.  Although this can be any "n.n.n.n" value, there is a convention established for how to use each version part (see the possible values description below).  Since this attribute is copied to the output data file it can, along with ConfigName, be used to uniquely identify the configuration file that was used to create a data file. 

This value also provides special functionality in the customizable GUI.  When a previously run scan is selected for viewing, it will compare the version of the data file with the current version of the configuration file.  If the configuration file is newer but the same major configuration version, then the tool will automatically reanalyze the existing data with the newer configuration.  This way previously collected data can have the latest rules processed against it so the data doesn't need to be gathered all over again.

Example Configuration Fragment

<Configuration
   ConfigVersion="1.6.7.3"
   ConfigName="ExBPA Master Input File"
   DownloadURL="http://www.microsoft.com/exchange/code/ExBPA"
   ArticleURL="http://go.microsoft.com/fwlink/?linkid=30939&amp;l={0}&amp;v=ExBPA.{1}&amp;id={2}"
   ChangeDescription="The changes made in this version are minor modifications to the configuration metadata."
>

Parent Element

Attribute Description
Configuration This element contains some general meta-information about the input configuration or the output data.
Run This output element contains general information about a particular run.

Possible Values

Value Description
"w.x.y.z"

This is the version of the configuration file.  Although this can be set to have any desired meaning, by convention it should be of the form “AppMajor.CfgMajor.CfgMinor.Private”.  This format should definitely be used if the customizable GUI is used so the functionality described above works properly. The conventional meaning of each part is:

  • AppMajor corresponds to the major version of the application that can process it.  The download new version logic will only download files that have the same AppMajor value as the application itself).
  • CfgMajor identifies the major version of the configuration file.  This indicates major structural changes have been made and reprocessing of old data files to apply new rules will only be done if the this version matches between the configuration and a data file.
  • CfgMinor is used to indicate a minor configuration change – usually the modification of or addition of rules or properties.
  • Private is used just for internal informational purposes.
