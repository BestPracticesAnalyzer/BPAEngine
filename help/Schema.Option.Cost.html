The Cost Attribute

This provides a hint to the application how long each object that matches the restriction is expected to take.  This can be used to come up with rough estimates of run time.  For example, in ExBPA the task restrictions determine how much to do against each server.  The ConnectivityTest option will only do a couple of brief checks against each server to determine if it has access, so the cost for this operation is much lower than the HealthCheck option, which will gather lots of data per server.

Example Configuration Fragment

<Option
   Name="Health"
   Display="Health Check"
   Description="Perform checks for common problems and misconfiguration."
   Cost="90,0"
   Includes="General,3"
/>

Parent Element

Element Description
Option This element identifies an option of a restriction.

Possible Values

Value Description
<any string> This exact meaning of this value is up to the implementation.  ExBPA interprets this as two comma-separated values.  The first value is an adjustable cost per server (in seconds); the second value is a fixed cost per server (also in seconds).  The adjustable cost will be affected by the network speed (such as a WMI query), the fixed cost is not (such as a perfmon sample).
