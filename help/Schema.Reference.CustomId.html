The CustomId Attribute

This is an identifier that can be set to allow a Customize element to override its contents.  This attribute is only needed for elements that are defined under a Type.

Example Configuration Fragment

<Reference 
   Name="NOTEPAD.exe File"
   Type="File"
   CustomId="NOTEPAD"
>

Parent Element

Element Description
Object This element contains information about a single item to be processed.
Reference This element refers to a Type element to be included in that position.
Rule This element contains information about a rule to execute on the output data.
Setting This element identifies a property to be retrieved from an Object.

Possible Values

Element Description
<any string> This is the name that can be used by a Customize element to alter this element.  Note that this name will have the names of the references prepended to it to keep it unique per reference.  The default value for this attribute depends on the type of element it is defined on:
  • For Object elements, this will default to the Name of the Object, if it exists.  If it does not, it will default to blank (and so the Object will not be customizable).
  • For Reference elements, this will default to the Name of the Reference.
  • For Rule elements, this will default to the Name of the Rule.  Note that the Rule Name also has the reference names prepended to it.  This will not be done twice when used as the default.
  • For Setting elements, this will default to the Key1 value of the Setting.
