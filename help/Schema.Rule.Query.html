The Query Attribute

This is an XPath expression that is evaluated, with the result placed in Result nodes under the Rule.  If this evaluates to true, a Message node will be generated.

Example Configuration Fragment

<Rule
   Name="fExPhysicalMachineUp"
   Query="$fClusterServiceRunning and not($szPSWinVer)"
   Error="Error"
   Sev="3"
   Title="Exchange cluster node could not be contacted"
   Text="Exchange cluster node {2} is down or unreachable. This could be due to a network or permissions problem."
   P2="$szExchangeNodeName"
   GUID="fa727882-9fdd-4664-9d49-f0ef95193b49"
   AlwaysEvaluate="True"
/>

Parent Element

Element Description
Rule This element contains information about a rule to execute on the output data.
Template This element identifies a rule template that can be applied to other rules so similar rules don't need to be defined separately.

Possible Values

Element Description
<any XPath statement> This will be evaluated to determine if the rule fires or not.  In addition to normal XPath, this statement can contain some specialty references:
  • "$." refers to the Value elements of the parent setting.  It is essentially identical to the XPath "../Setting/Value" but provides a little easier data entry.
  • "$<rule name>" refers to the Value elements under the Result element of the named rule.  This is very useful to avoid having to create complex path statements that would have to be changed whenever the configuration hiearchy is modified.  It is recommended that this notation be used in place of path statements whenever possible.
  • <function> refers to one of the built-in or extended functions supplied.  See Function Processors for a list of built-in functions.
