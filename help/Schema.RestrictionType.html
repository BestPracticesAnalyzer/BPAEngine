The RestrictionType Element

This element identifies a restriction that can be applied to the configuration on a run.  It's used by the restriction built-in configuration preprocessor.  See Configuration Preprocessors for more details.

Example Configuration Fragment

In the configuration file:

<RestrictionType
   Name="Task"
   Display="Task"
   Description=""
   Collapse="False"
>
     <Option
         Name="Health"
         Display="Health Check"
         Description="Perform checks for common problems and misconfiguration."
         Cost="90"
         Includes="General,3"
     />
     <Option
         Name="ConnectivityTask"
         Display="Connectivity Test"
         Description="Perform a quick connection test to validate network access and permissions."
         Cost="60"
         Includes="Connectivity"
     />
     <Option
         Name="BaselineTask"
         Display="Baseline"
         Description="Compare settings against user-specified values."
         Cost="45"
         Includes="CustomBaseline"
      />
</RestrictionType>

Parent Elements

Element Description
Configuration Restriction elements only exist in the input configuration.

Child Elements

Element Description
Option This element identifies an option of a restriction.

Attributes

Attribute Description
Collapse This indicate whether the GUI should hide elements that are only collected because they are ancestors of elements that pass the restriction.
Description This is an informational description of the purpose of this restriction.
Display This is the value that should be displayed through the UI when presenting the user a choice of options for this restriction.
Name This is the unique name of the restriction.
