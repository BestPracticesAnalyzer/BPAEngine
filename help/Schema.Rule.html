Best Practices Analyzer Core Library
The Rule Element

This element identifies a rule to be evaluated against the collected date.  All rules are processed in the analysis phase, which takes place after the data collection phase completes.  A rule can therefore reference settings from anywhere in the hierarchy without worrying about order.  Rules can also reference the results of other rules, in which case the engine determines the proper sequence of rule evaluation. 

Example Configuration Fragment

In the configuration file:

<Rule
   Name="fExchangeServerNameRegMatch"
   Query="not ($fClusterServiceRunning) and not ($fExchangeServerNameTooLong) and $.!=$szTCPHostname"
   Error="Error"
   Sev="2"
   Title="Server name mismatch"
   Text="There is a discrepancy with the Exchange server name {2} between the registry and Active Directory. The computer may have been renamed or third-party clustering software may be running. Hostname in registry: {1}."
   P1="$szTCPHostname"
   P2="$szExchangeServerName"
   GUID="13cf95ee-9562-4f73-b65c-0425ae06424b"
/>

In an output data file:

Same as above (rules are copied as is to the output).

Parent Elements

Element Description
Reference This element refers to a Type element to be included in that position.
Setting Rule elements are nested under settings.  Although it is not necessary for the rule to actually deal with the parent setting, that is usually the case.

Child Elements

Element Description
none On input, Rules have no child elements.
Result This, on output, is the result of the rule. 

Attributes

Attribute Description
AlwaysEvaluate If this is set, this pivot will be evaluated even if rules it refers to are not run.  Otherwise such pivots are skipped.
CustomId This is an identifier that can be referenced by a Customize element to override its properties.  This is only meaningful if the Rule is nested under a Type.  This defaults to the Name of the Rule.
Error This indicates the error level assigned to this rule.
EventID This indicates the event ID to be generated when this rule is fired.  This is used by the events Issue Processor.
GUID This GUID points to a detailed article that explains this rule.
Name This is a unique identifier of the rule, composed of only letters and digits.
Pass This is generated on output and indicates whether the rule evaluated to true or false.
P0 - P9 This parameters may be referenced by the Query.
Query This is the XPath expression to evaluate to determine if the rule fires.
<restriction> This is a custom-defined restriction attribute.
S0 - S9 These fill in the parameters in the Text string.
SDMName This provides a hint to be used during conversion of a BPA Object element to an SDM element.
Sev This is a severity rating to be used when the rule fires. 
Template This identifies a Template that can be used to define a reusable set of Rule attributes.
Text This is the text to be displayed when the rule fires. 
Title This is the title of the issue for display purposes.
