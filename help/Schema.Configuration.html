Best Practices Analyzer Core Library
The Configuration Element

This element is the root element of the metadata information for an input configuration or output data file.  It contains general information about the file.  On output, it will contain information from the most recent run that processed the file.

Example Configuration Fragment

In the configuration file:

<Configuration
   ConfigVersion="1.6.7.3"
   ConfigName="ExBPA Master Input File"
   DownloadURL="http://www.microsoft.com/exchange/code/ExBPA"
   ArticleURL="http://go.microsoft.com/fwlink/?linkid=30939&amp;l={0}&amp;v=ExBPA.{1}&amp;id={2}"
   ChangeDescription="The changes made in this version are minor modifications to the configuration metadata."
>

In an output data file:

<Configuration
   CustomerName=""
   StartTime="09/30/2004 23:20:39"
   ObjectsTracked="9"
   Completed="OK"
   ConfigVersion="1.5.7.1"
   Operations="analyze"
   Label="Demo Data File"
>

Parent Elements

Element Description
root It's only parent is the root most element of the XML.

Child Elements

Element Description
ConfigPreprocessor This element identifies a managed class that will be used to preprocess the configuration.  This is not yet implemented.
ExtFormat This element maps an Setting.Format to a managed class.
ExtFunction This element maps a custom rule function to a managed class.
IssueProcessor This element identifies a managed class that will be invoked when a rule fires.
ObjectProcessor This element maps an Object.Type to a managed class.
RestrictionType This element defines a restriction that can be applied to a run.
Run This output element contains general information about a particular run.
Substitution This element contains a parameter value that can be references elsewhere in the configuration.
Template This element identifies a rule template that can be applied to other rules so similar rules don't need to be defined separately.

Attributes

Attribute Description
ArticleURLThis is the root URL to use for detailed article look ups (using the more info links).
ChangeDescriptionThis contains some text that indicates what the latest changes are in this configuration.
CompletedThis is an output attribute that indicates the results of the last scan done.
ConfigNameThis is name of the configuration file.
ConfigVersionThis is the version of the configuration file.
DownloadURL This is where to look for configuration and binary updates.
LabelThis is an output attribute that can be set to logically identify the contents of a data file.
ObjectsTrackedThis is an output attribute that indicates how many objects were marked for tracking in the latest run.
ReferencedFilesThis is a comma separated list of file names that are given in Reference elements.
StartTimeThis is an output attribute that indicates the time the last run was begun.
