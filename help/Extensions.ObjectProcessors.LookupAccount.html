Best Practices Analyzer Core Library
The LookupAccount Object Processor

This method will perform a lookup of the given account name and retrieve the SID associated with it, or the other way around.

There will be zero or one Instance elements created, depending on whether the lookup succeeded.  The Name attribute for a successful instance will be the account name or sid supplied.  For each Setting requested, a Value element will be generated from the corresponding account data retrieved.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="LookupAccount"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.LookupAccountObjectProcessor"
/>

An example object definition:

<Object
   Type="LookupAccount"
   Key1="Name"
   Key3="%NBDOMNAME%\Administrator"
>
     <Setting Key1="Sid"/>
</Object>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1

This is the type of lookup to perform.  It can have one of the following values:

  • "Name" uses Key3 as an account name and it will retrieve account information for it.
  • "Sid" uses Key3 as a string SID and it will retrieve account information for it.
Key2 This is the system to perform the lookup against.
Key3 This is the account name or the SID, depending on the value of Key1.
Type This must be set to "LookupAccount".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1

This is the value to retrieve.  It can be:

  • "Domain" is the domain name the account is in.
  • "Name" is the name of the account.
  • "Sid" is a string version of the SID.
  • "Type" is the type of the account.
