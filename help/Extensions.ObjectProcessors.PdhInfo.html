Best Practices Analyzer Core Library
The PdhInfo Object Processor

This is used to break up a previously collected performance file into smaller chunks for more efficient and accurate processing.  Such a file can be created with the PdhCollect object processor.

This will create Instance elements for each server and time interval in the blg file.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="PdhInfo"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.PdhInfoObjectProcessor"
/>

An example object:

<Object
   Type="PdhInfo"
   Key1="%SRVNAME%"
   Key2="600"
   Key3="datafile.blg"
>
     <Setting Key1="StartTime"/>
     <Setting Key1="EndTime"/>
     <Setting Key1="Duration"/>
</Object>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1 The name of the server to access.  If processing an already created blg file, this can also be set to "*", in which case it will process all servers in the file.
Key2 This is the desired segment duration, in seconds.  Each segment of that duration will be broken into a separate instance.
Key3 The name of the blg file to process.
Type This must be set to "PdhInfo".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1 This can be one of the following:
  • Duration.  This is the length of the segment, in seconds.
  • EndSample.  This is the ending sample index.
  • EndTime.  This is the ending time of the segment.
  • Server.  This is the name of the server.
  • StartSample.  This is the starting sample index.
  • StartTime.  This is the starting time of the segment.
