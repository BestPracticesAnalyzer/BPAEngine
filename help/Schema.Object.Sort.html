The Sort Attribute

This indicates how the instances should be ordered in the output.

Example Configuration Fragment

<Object
   Type="WMI"
   Key3="Win32_NTLogEvent"
   Key4="Logfile='Application' and EventType=1"
   Sort="TimeWritten"
>

Parent Element

Attribute Description
Object Sort attributes are specified on input Object elements.

Possible Values

Value Description
"<property> [ASC | DESC] <key>" This indicates how the instances are to be ordered:
  • <property> defines the property name to use to do the ordering.
  • ASC indicates the order should be ascending.  The ASCENDING keyword also works.  This is the default.
  • DESC indicates the order by be descending.  The DESCENDING keyword also works.
  • <key> is the Key attribute to use on each Setting to get the property name from.  This defaults to Key1 (which is what most object processors require the property name to be specified in).

Whenever an instance is processed, the engine will look through all the Setting elements for it and find the one that has the value of the <key> attribute set to <property>.  It will then extract the first Value of that Setting and use that value to insert the instance in the correct position.  Note that all values will be treated as strings.

If this attribute is not given, the order of instances will be determined by the data source.
