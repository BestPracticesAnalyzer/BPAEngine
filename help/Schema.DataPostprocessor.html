Best Practices Analyzer Core Library
The DataPostprocessor Element

This element identifies a data postprocessor that will be invoked after data collection and analysis.  These postprocessors can go through the data document and modify it as needed while still retaining a parsable format, or it can just examine the contents.  The order these elements are put in will determine the order they are executed in. 

See the Data Postprocessors section for more details.

Example Configuration Fragment

In the configuration file:

<DataPostpreprocessor
   Name="0 SQM Postprocessor"
   Assembly="ExBPA.Shared.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Exchange.SQMDataPostprocessor"
/>

Parent Elements

Element Description
Configuration DataPostprocessor elements only exist in the input configuration.  They are not echoed to the output data file.

Child Elements

Element Description
none This element has no children.

Attributes

Attribute Description
Assembly This should point to a DLL that contains a C# assembly which has one or more configuration preprocessors defined in it.
Class This is the name of a class that has implemented the ConfigPreprocessor logic.
Name This identifies the particular data postprocessor. It also servers to determine the sequence the postprocessors will be executed in (alphabetically by this value).
ObjectType This attribute means the same as Name, but is discontinued.
