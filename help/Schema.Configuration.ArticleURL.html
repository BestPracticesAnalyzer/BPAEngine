Best Practices Analyzer Core Library
The ArticleURL Attribute

This indicates the root URL name that should be used to access detailed articles from the web.  This is only needed if an application is supporting detailed article access in its UI.  The customizable UI supplied by the library does support this by default but it can be disabled (this is controlled by BPACustimizations.AllowDetailedArticleLinks).  If enabled, it will use the value of this attribute to prefix the URL used to access the web articles.  This URL is only used if web connectivity is detected.  If no web access is found and this is still enabled, the customizable UI will direct "Show Me More" links to the local help file instead.

Example Configuration Fragment

<Configuration
   ConfigVersion="1.6.7.3"
   ConfigName="ExBPA Master Input File"
   DefaultLevel="3"
   DownloadURL="http://www.microsoft.com/exchange/code/ExBPA"
   ArticleURL="http://go.microsoft.com/fwlink/?linkid=30939&amp;l={0}&amp;v=ExBPA.{1}&amp;id={2}"
   ChangeDescription="The changes made in this version are minor modifications to the configuration metadata."
>

Parent Element

Attribute Description
Configuration This element contains some general meta-information about the input configuration or the output data.

Possible Values

Value Description
URL string with three parameters For the customizable UI, this URL is expected to take three parameters (using .NET string format style, i.e "{0}").  The first parameter will be replaced with the locale of the system, second parameter will be replaced with the major version number of the application, and the third parameter will be replaced with the GUID of the particular issue being looked up.  Note the UI also verifies that this URL name starts with "http://go.microsoft.com" to avoid possible spoofing issues.
<any> If a separate UI implementation is used, this can be any format desired (or not used).
