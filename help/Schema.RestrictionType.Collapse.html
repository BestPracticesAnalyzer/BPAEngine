The Collapse Attribute

This indicates whether the build in restriction configuration preprocessor (see Configuration Preprocessors) is supposed to complete eliminate non-matching elements are just mark them as Deferred when processing this restriction.

Example Configuration Fragment

<RestrictionType
   Name="Task"
   Display="Task"
   Description=""
   Collapse="False"
>

Parent Element

Element Description
RestrictionType This element identifies a restriction to apply to the configuration.

Possible Values

Value Description
"True" Elements that are only processed because they are directly above a matching element should be marked with a Display="Hide" This has the effect that the GUI will present matching elements at the root level of the Detailed View.
<anything else> All processed elements are displayed normally.
