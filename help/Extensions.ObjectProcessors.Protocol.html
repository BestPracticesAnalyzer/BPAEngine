Best Practices Analyzer Core Library
The Protocol Object Processor

This is used to measure whether the internet protocols are working properly on the remote server or not.

This will only create one Instance element on output.  The Name attribute for the instance is taken from the object.  For each Setting specified, two Value elements will be created.  The first will contain the response time, in milliseconds.  The second will contain the actual response received.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="Protocol"
   Assembly="BPA.NetworkCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.ProtocolObjectProcessor"
/>

An example object:

<Object
   Type="Protocol"
   Name="SMTP Open Relay Test"
   Key1="%SRVNAME%"
   Key2="TCP"
   Key3="%SMTPPORT%"
   Key4="60000"
   Timeout="%300"
>
     <Setting Key1="Connect"/>
</Object>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1

This name of the server to access.
Key2 The communication protocol, such as "TCP", "UDP" or "ICMP".
Key3 The remote port number, such as 143 for IMAP.
Key4 Time out value for the connection.
Type This must be set to "Protocol".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1 The command to execute, such as "Connect".
Key2 If set to "SkipRead", this indicates we should not attempt to read a response after the command is sent. Such settings will not have Value elements created for them.
