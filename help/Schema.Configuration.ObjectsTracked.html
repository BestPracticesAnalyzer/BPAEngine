Best Practices Analyzer Core Library
The ObjectsTracked Attribute

The returns the number of instances of Object elements that had TrackProgress enabled for them in the run.  This can be used for informational purposes.  For example, ExBPA tracks progress on each Exchange server it analyzes.  So this count becomes the number of servers processed in the run (the count is actually one greater than that because it also tracks progress on the whole run).

Example Configuration Fragment

<Run
   ConfigName="ExBPA Master Input File"
   ConfigVersion="1.5.6.1"
   AppVersion="1.0.7408.1"
   StartTime="09/30/2004 23:20:39"
   Timeout="240"
   Threads="25"
   Operations="collect,analyze"
   DC="CONTOPGC01"
   EndTime="09/30/2004 23:25:48"
   Completed="OK"
   ObjectsTracked="9"
>

Parent Element

Attribute Description
Configuration This element contains some general meta-information about the input configuration or the output data.
Run This output element contains general information about a particular run.

Possible Values

Value Description
<number> This is a count of instances that had TrackProgress enabled.
