Best Practices Analyzer Core Library
The Label Attribute

This attribute is an information string that associates an identifiable label with a data file.  The customizable GUI will display this label in its select scan screen.  Note that this is put on the Configuration element in the output data file but not on the Run element (this is because it can't be changed from run to run). 

Example Configuration Fragment

 <Configuration
   CustomerName=""
   StartTime="09/30/2004 23:20:39"
   ObjectsTracked="9"
   Completed="OK"
   ConfigVersion="1.5.7.1"
   Operations="analyze"
   Label="Demo Data File"
>

Parent Element

Attribute Description
Configuration This element contains some general meta-information about the input configuration or the output data.

Possible Values

Value Description
<any> This can be any descriptive string.  It does not need to be unique, but it will help a user to distinguish between data files if it is.
<not present> This attribute is optional.  If it is not given, the customizable GUI will display the start time of the scan instead.
