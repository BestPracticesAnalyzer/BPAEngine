The P0 - P9 Attributes

These are XPath expression parameters to be evaluated and filled into the corresponding format items in the Text attribute if a Message is generated.  Note that you should not have both an Sn and a Pn attribute for the same parameter.

Example Configuration Fragment

<Template
   Name="fBaseline"
   Error="Baseline"
   Sev="2"
   Text="The value of setting {4} on server {2} is '{9}', which differs from the specified value of '{8}'"
   P2="$szExchangeServerNameCaps"
   P4="../@DisplayName"
   P9="$."
/>

Parent Element

Element Description
Rule This element contains information about a rule to execute on the output data.
Template This element identifies a rule template that can be applied to other rules so similar rules don't need to be defined separately.

Possible Values

Element Description
<any XPath expression> This expressions is processed and then replaces the corresponding parameter in the Text attribute (for example, P0 will replace "{0}").
