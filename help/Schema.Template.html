Best Practices Analyzer Core Library
The Template Element

This element provides a standard set of attributes that can be referenced rules.  This can be used to avoid redundancy. 

Example Configuration Fragment

In the configuration file:

<Template
   Name="tVirusConfig"
   Query="false()"
   Title="Anti-Virus configuration"
   Text="{0} is installed on server {2} but file exclusions have not been properly configured."
/>

Parent Elements

Element Description
Configuration This element contains some general meta-information about the input configuration or the output data.

Child Elements

Element Description
none This contains no child elements.

Attributes

Attribute Description
AlwaysEvaluate If given, this is copied to any rules that reference it (unless the rule already defines this attribute).
Error Same as above.
EventID This indicates the event ID to be generated when this rule is fired.  This is used by the events Issue Processor.
GUID Same as above.
Name This is the unique name used to reference the template. Rules or other templates with their Template attribute set to this will copy its attributes.
Pass If given, this is copied to any rules that reference it (unless the rule already defines this attribute).
P0 - P9 Same as above.
Query Same as above.
<restriction> Same as above.
S0 - S9 Same as above.
