The Substitution Attribute

This creates a variable whose value will be set to the value of the setting.  If the setting is multi-valued, this will be set to the first occurrence.  Other settings and objects (nested beneath this setting’s parent object) can reference this variable by putting percent signs around the name.  This can be used where the value of a setting contains data needed elsewhere.  For instance, the “name” property of an AD object that represents an Exchange server will be the name of the server.  WMI objects need to reference server names to work, so they would reference a substitution defined by an object name.

Understanding substitution attributes and using them appropriately is a key to unlocking the power of a BPA configuration.

Substitutions can also be defined globally using the Substitution element.

Example Configuration Fragment

<Setting
   Key1="StartName"
   Substitution="CLUSVCNAME"
   Retrieve="FailHard"
>

Parent Element

Element Description
Setting This element identifies a property to be retrieved from an Object.

Possible Values

Element Description
<any string> This is the name of the variable to assign the value of this setting to.
