Issue Processors

Issue processors are used as a hook into when the engine fires a Rule.  For each rule fired, each issue processor will be called and given a chance to take action based on that rule.  An alternative method of dealing with this is to wait for the analysis phase to complete, and then parse out all the Message elements from the resulting document.  Which option you choose is up to you.  Issue processors are defined use IssueProcessor elements in the configuration.

A couple of different issue processors have been defined:

Message Generation This is the basic function that causes a message to be generated when a rule fires.  This message will be created as a Message element in the output data file.  This issue processor is built-in and so does not need to be defined in the configuration.
Events This will cause an event log message to be created for each rule that fires and has an EventID attribute defined for it.
