Best Practices Analyzer Core Library
The File Object Processor

This object processor can be used to collect data from a file or from a set of files.  It can get a few specific properties from flat file or directory, and it can also extract entries from an INI file or an XML file.

This will create one Instance element in the output file per file found by the specified query.  The Name attribute for an instance will be the file name.  Each Setting will have a Value element added to it that contains the value found for that property.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="File"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.FileObjectProcessor"
/>

An example object:

<Object
   Type="File"
   Key1="%SRVNAME%"
   Key2="%EXCHBINPATH%"
>
     <Setting
         Key1="NTSD"
         Format="SecurityDescriptor"
         NotForExport="True"
     />
</Object>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1

This is the server to access the file from.
Key2 This is the name of a file or a directory (absolute path).
Key3 This is a file matching expression that will match any file names in the given directory.  This attribute will be ignored if Key2 specifies a file.
Key4

This is a search scope to use.  It can have one of the following values:

  • "Base" is just the exact directory or file given.  This is the default if Key3 is not specified.
  • "OneLevel" means to match entries directly under the given directory only.  This is the default if Key3 contains a value.
  • "Subtree" means to match against all entries in the directory and any subdirectories.  Note: this is not implemented yet (this currently behaves the same as OneLevel).

This attribute will be ignored if Key2 specifies a file.
Key5 This identifies what types of entries to include in the output:
 
  • "Files" means only files are to be output as instances.  This is the default.
  • "Directories" means only directories are to be output as instances.
  • "All" means both files and directories are to be output as instances.

This attribute will be ignored if Key2 specifies a file.
Type This must be set to "File".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1

The following values are recognized:

  • "Files" gets the list of files for the directory specified by Object.Key2.  Each file name will be returned in a separate value.  Note: this usage is being deprecated, but it will still work with a directory instance, but not a file instance. 
  • "FullName" gets the full path name to the file or directory.
  • "Ini" will treat the file as a .ini file and Key2 and Key3 will indicate what values to retrieve.  This will not work with a directory instance.
  • "LastModifiedTime" retrieves the time the file or directory was last written to.
  • "NTSD" retrieves the security descriptor for the file or directory.  Format="SecurityDescriptor" should be specified on this setting to assist displaying the data.
  • "Version" retrieves the version stamp on the file.  This will not work with a directory instance.
  • "MD5" retrieves the md5 checksum on the file.  This will not work with a directory instance.
  • "SHA1" retrieves the sha1 checksum on the file.  This will not work with a directory instance.
  • "Xmlprop" will treat the file as a .xml file and Key2 and Key3 will indicate what values to retrieve.  This will not work with a directory instance.
Key2 If Key1 is set to "Ini", this will be the appname to lookup in the file.  If Key1 is set to "Xmlprop", this will be an XPath query to perform on the contents of the specified file.  If Key1 is set to "Files", this is the pattern of file names to match.  For other values of Key1, this attribute is not used.
Key3 If Key1 is set to "Ini", this will be the propname to lookup in the file.  If Key1 is set to "Files", this can specify an optional "Generation" hint which causes the processor to treat the files as Jet log files and find the last log generation from the names (this functionality is rather specific to Exchange).  For other values of Key1, this attribute is not used.
