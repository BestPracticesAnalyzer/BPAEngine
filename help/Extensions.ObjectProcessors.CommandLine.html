Best Practices Analyzer Core Library
The CommandLine Object Processor

This will launch a command line executable and wait for it to finish.  Note: this can only be used to launch an executable on the local machine.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="Command"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.CommandLineObjectProcessor"
/>

An example object:

<Object
   Type="Command"
   Key2="cmd"
   Key3="dir /b c:\"
>
     <Setting Key1="Output" Key2="stdout"  />

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key2 This is the executable to run. 
Key3 This specifies the command line parameters to pass to the executable.
Type This must be set to "Command".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1 This is the command key that describes how it should deal witht he output.  It can be one of the follow values:
  • Output.  This will retrieve the data written to the output stream specified in Key2.
  • Regex.  This will process the regular expression defined in Key2 against the output stream specified in Key3.
  • ReturnValue.  This will retrieve the value returned by the executable.
Key2 The meaning of this attribute depends on the value of Key1.  If Key1 is "Output", this can be either "stderr" or "stdout" (with the latter being the default).  If Key1 is "Regex", this should be the regular expression to process against the output stream specified in Key3.  If Key1 is "ReturnValue", this attribute is not used.
Key3 If Key1 is "Regex", then this can be either "stderr" or "stdout" (with the latter being the default).  If Key1 is anything else, this attribute is not used.
