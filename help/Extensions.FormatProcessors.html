Best Practices Analyzer Core Library
Format Processors

Format processors are used to convert retrieved data to a usable format.  Everything the BPA engine deals with is a string, so these extensions allow for converting binary or other types of data to strings for further use.  These processors are defined using ExtFormat elements in the configuration and they are referenced through the Format and SubstitutionFormat attributes on Setting elements.

A number of built-in format processors are supplied in the SDK (these do not actually use the extension logic so they do not need to be defined in the configuration, but the concept is the same):

Sid This will convert a binary SID to a string SID if specified.
SecurityDescriptor This currently defaults to the same functionality as System.Byte[] when storing the data.  When rendering the data, the security descriptor is presented as a set of properties and aces.
System.Byte[] This will display any value as a byte string, which will simply be of the form nnnnnn..  where each nn pair is the hex value of a byte.
System.DateTime This display a timestamp property in GMT.
System.DateTime.Local This display a local timestamp property in GMT.
System.Guid This converts a byte array into a displayable GUID of the format xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.  Note: if something is a GUID type to begin with, this will be its default display format.  This was created if the setting was mistyped in the data store it is being retrieved from (this is the case with some properties in the Active Directory).
.* This will perform a regex transformation on the given data, which is expected to be a string.  The content of the Format attribute on a Setting is expected to be “<match>----<modify>” where the first string is a regular expression that matches values in the data string, and the second string is a regular expression that indicates how that data should be modified.  The four dashes are used to separate the two expressions.

These additional format processors use the extension logic and so need to be configured to be accessible:

AverageFormat This will return an average of all the numeric values for the Setting.  It is designed to work in conjunction with the Pdh object processor to reduce the amount of data that needs to be stored in the XML.
LowerCase This will convert all text to lowercase.
MaxFormat This will return the maximum value of all the numeric values for the Setting.  It is designed to work in conjunction with the Pdh object processor to reduce the amount of data that needs to be stored in the XML.
MinFormat This will return the minimum value of all the numeric values for the Setting.  It is designed to work in conjunction with the Pdh object processor to reduce the amount of data that needs to be stored in the XML.
MinNonZeroFormat This will return the minimum non-zero value of all the numeric values for the Setting.  It is designed to work in conjunction with the Pdh object processor to reduce the amount of data that needs to be stored in the XML.
NineteeithPercentileFormat This will return the value at the nineteeith percentile of all the numeric values for the Setting.  It is designed to work in conjunction with the Pdh object processor to reduce the amount of data that needs to be stored in the XML.
SigmaFormat This will return the sigma value of all the numeric values for the Setting.  It is designed to work in conjunction with the Pdh object processor to reduce the amount of data that needs to be stored in the XML.
