Best Practices Analyzer Core Library
Data Postprocessors

Data postprocessors can be used to manipulate or examine the data document after all the content has been collected and analyzed.  While this could also be done by the executable wrapper once the engine completes a scan, using this extensions makes it easier to create reusable functionality.  These are defined using DataPostprocessor elements and they will be executed in the order they are defined.  Each implementation can do whatever it likes to the data file as long as the result conforms to the BPA schema.

The currently built-in extensions that use this are:

SQM This postprocessor will go through a data file and process special SQM attributes to generate SQM data.  Configuration information for the SQM postprocessor:

<DataPostprocessor
   Name="0 SQM Postprocessor"
   Assembly="BPA.Common.dll"
   Class=
"Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Common.SQMConfigPreprocessor"
/>

When this is defined, the following attributes have meaning:

  • Configuration.SQMUploadURL.  This identifies the URL where SQM data should go.
  • Object.SQM.  If this is set to "PerInstance", all SQM data generated under this object will be done in a session per instance.
  • Setting.SQM.  This can be a data point (integer) followed by a SQM operation type (with a semicolon separating them).  The operation type can be: "Set", "Increment", "SetIfMax", "SetIfMin" and "AddToAverage".

To use this, you will need to get the latest custsat.dll from the SQM team.  

EdisonSQM This postprocessor will go through a data file and process special SQM attributes to generate SQM data.  This is designed to be used when BPA is run under the Edison framework.  Configuration information for the EdisonSQM postprocessor:

<DataPostprocessor
   Name="0 Edison SQM Postprocessor"
   Assembly="BPA.Common.dll"
   Class=
"Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Common.EdisonSQMConfigPreprocessor"
/>

When this is defined, the following attributes have meaning:

  • Configuration.SQMSet.  This int identifies the set id to use for all data collected.
  • Configuration.SQMVersion.  This int identifies the version to use for all data collected.
  • Rule.SQM.  This identifies the data point (int) to use for this rule.  Rules will be marked either as pass or fail.

To use this, you will need to get the latest McdBpaSqmWrapper.dll and McdBpaSqmCWrapper.dll from the Edison team.
