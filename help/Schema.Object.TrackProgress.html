The TrackProgress Attribute

This indicates that the engine should track the progress of this object and provide status updates as processing occurs.  The status updates are sent via the ExecutionOptions.Status callback.  Whenever a new object is processed underneath an object with this attribute specified, the engine will issue the callback, providing the name of the parent object being processed and the percentage complete based on the number of objects left to process. 

Note that this percentage may actually get lower on some occasions as additional instances are created for some objects.  For instance, if an object being tracked has 10 objects underneath it and one has been processed (and resulted in a single instance being created), the completion percentage will be 10%.  If the second object has two other objects under it, and processing it results in 11 instances being created, the total number of objects to process under the tracked object will rise from 10 to 30, so the percentage will fall to 2/30 or 7%.  This percentage only takes into account the total number of objects to process - it does not account for estimated times for processing each object.  So this can only provide a hint to the user to indicate how far along things are.  Accuracy is indeterminate.

The customizable GUI uses these status messages to present progress bars for each object being tracked as well as an overall progress bar for the whole run.  

Example Configuration Fragment

<Object
   Type="Group"
   Name="Full Run"
   TrackProgress="TrackProgress"
>

Parent Element

Element Description
Object These attributes are specified on input Object elements.

Possible Values

Value Description
"TrackProgress" This object should have status callbacks issues to track its progress as it is being processed.  Note: by convention, if the Name of the object is "Full Run", then a GUI may consider this to contain status for the run as a whole rather than an individual object.
"TrackFailHard" Same as above, but also indicates that a hard failure on this entry should result in an abort progress tracking status update being sent.
<not present> No progress tracking is to be done.
