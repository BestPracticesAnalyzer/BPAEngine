Best Practices Analyzer Core Library
The Count Attribute

This attribute identifies a property to count over.  When this is given, all settings will be ignored by the object processor and it will get all instances of the given object and for each instance it will look at this property only for each unique value found it will count how many instances match that value.  A typical use would be to determine all the types of clients accessing a server and how many of each there are.  Note: this implementation is up to the object processor to implement.  Many of the object processors supplied with this library do not support this feature.  See AddCountSettingNodes and CollectCountPropTotals methods for some helper functions that assist an object processor with this functionality.

For each unique value of the counting property, a Setting element with one Value element will be created in the output data file.  Key1 of the Setting element will have a value of the counting property.  The Value element underneath it will contain an integer count of the number of instances that were found and that had the counting property value equal to the value of Key1.

Example Configuration Fragment

<Object
   Type="WMI"
   Key1="%SRVNAME%"
   Key2="root\MicrosoftExchangeV2"
   Key3="Exchange_Logon"
   Key4="ServerName='%EXSRVNAME%' AND StorageGroupName='%SGNAME%' AND StoreName='%MDBNAME%'"
   Count="ClientMode"
   Timeout="%300"
/>

Parent Element

Element Description
Object Async attributes are specified on input Object elements.

Possible Values

Element Description
<property name> This identifies the property to count over.
"*" This indicates the object processor should just count the total number of instances.  In this case, only one Setting element will be created and its Key1 value will be set to "*" and the value of the Value element underneath it will be the total instance count.
