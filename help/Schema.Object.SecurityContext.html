The SecurityContext Attribute

This identifies what security context should be applied to when processing this object.  Security contexts can be mapped to credentials (account name, domain, and password) through ExecutionOptions.CredentialsList.  When an object is about to be dispatched, the engine checks this attribute.  If the attribute exists, and it is different than the current security context, and the context exists in the run time credentials list, then the engine will impersonate the credentials prior to dispatching the object.  This way, that object will be processed with the specified credentials rather than the default.  ExBPA also supports empty string ("") as security context which can be used to revert to the default process security credential if necessary. ExBPA uses this to allow Active Directory queries to use one set of credentials, and Exchange server queries to use another (many companies don't have a single account that can access both).

Example Configuration Fragment

<Object
   Type="Group"
   Name="Exchange Organization"
   SecurityContext="EXLOGON"
>

Parent Element

Element Description
Object These attributes are specified on input Object elements.

Possible Values

Value Description
<any string> This should be a short descriptive name of the meaning of this security context.  This name will be mapped to a user account at run time by the calling application.
