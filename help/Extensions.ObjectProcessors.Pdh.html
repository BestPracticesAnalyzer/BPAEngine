Best Practices Analyzer Core Library
The Pdh Object Processor

This is used to do simple performance counter sampling over time using the Performance Data Helper library.  It can execute the performance data gathering in real time, or it can process data from an already created blg file (such a file can be created with the PdhCollect object processor.

This will create Instance elements for each unique category/instance pair introduced by the settings (including wildcard expansion).  It can also create Instance elements per time range when processing an existing blg file.  The Name attribute for these instances will read "Category(Instance)".  Each instance will have an extra attribute stamped on it of "SampleTime" which will indicate when the sample was begun.  Each Setting will have one Value element added, and the value will be a comma-separated list of counter values taken at each sample interval.

Example Configuration Fragment

The ObjectProcessor definition:

<ObjectProcessor
   ObjectType="Pdh"
   Assembly="BPA.ConfigCollector.dll"
   Class="Microsoft.WindowsServerSystem.BestPracticesAnalyzer.Extensions.PdhObjectProcessor"
/>

An example object:

<Object
   Type="Pdh"
   Key1="%SRVNAME%"
   Key2="10"
   Key3="10"
>
     <Setting Key1="Processor" Key2="% Processor Time" Key3="*"/>
     <Setting Key1="Processor" Key2="% Privileged Time" Key3="*"/>
     <Setting Key1="Processor" Key2="% User Time" Key3="*"/>
     <Setting Key1="System" Key2="Context Switches/sec" Key3=""/>
     <Setting Key1="System" Key2="System Calls/sec" Key3=""/>
</Object>

Specific Object Attributes (see Object for a full attribute list)

Attribute Description
Key1 The name of the server to access.  If processing an already created blg file, this can also be set to "*", in which case it will only process the first server found in the file.
Key2 The number of samples to take.
Key3 When collecting live data, this is the period between samples (in seconds).  When processing from a blg file, this is the file name.
Key4 When processing from a blg file, this can be the starting sample number, or it can be a time range in the format <date/time string>-<data/time string>.  This parameter is ignored when collecting live data.
Key5 When processing from a blg file, this can be the name of a cached query to create a use (if the query does not yet exist, it will be created and saved under that name, if it already exists, it will be reused).  Also, this can be of the form "ResetCache: <query name>" in which case it will clear that query, or it can just be "ResetCache", in which case it will clear all queries.  This parameter is ignored when collecting live data.
Type This must be set to "Pdh".

Specific Setting Attributes (see Setting for a full attribute list)

Attribute Description
Key1 The Perfmon counter category.
Key2 The Perfmon counter name.
Key3 The Perfmon counter instance. It may contain wildcards.

Note: to reduce the amount of information saved in the XML due to large performance samples, use the AverageFormat, MinFormat, MaxFormat, NineteeithPercentileFormat, SigmaFormat, or MinNonZeroFormat if appropriate.
