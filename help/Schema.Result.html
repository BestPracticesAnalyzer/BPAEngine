Best Practices Analyzer Core Library
The Result Element

This element identifies the result of a rule evaluation.  It is only created in an output data file.  It does not exist in the input configuration.

The inner text of this element is the result of evaluating the Query of the parent Rule.  If the Query returns an XPath-type boolean, string, or number, a Rule will have a single Result node containing only text.  If it returns a node-set, there may be multiple Value elements under the Result element, each containing one value from the node-set.  The Result of a Rule may be referenced by the Pn and Query attributes of other Rule nodes.

Example Configuration Fragment

In an output data file:

<Rule
   Name="szExchangeServerNameCaps"
   Query="upper-case($.)"
   Pass="Unknown"
>
     <Result>EXAMER01</Result>
 

Parent Elements

Element Description
Rule Result elements are nested under the Rule that generated them.

Child Elements

Element Description
none If the Rule evaluation does not result in a node-set, then there will be no children and the inner text of the Result element itself will be the data.
Value If the Rule evaluation results in a node-set, there will be one of these elements per node.

Attributes

Attribute Description
none This element has no attributes.
