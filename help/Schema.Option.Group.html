The Group Attribute

This provides a hint to the application about how the options are grouped within a restriction. 

Example Configuration Fragment

<Option
   Name="Perf"
   Display="Performance Check"
   Description="Perform some general performance checks."
   Cost="0,7200"
   Includes="Performance,5"
   Group="Health"
/>

Parent Element

Element Description
Option This element identifies an option of a restriction.

Possible Values

Value Description
<any string> This exact meaning of this value is up to the implementation.  In ExBPA, all scan type options that do not have a group are considered exclusive - you can select one and only one of them for a given run.  The health check option, however, can include a number of optional checks such as doing additional performance checks, checks on recipients, and checks on permissions.  These are optional because they may take some additional time to process.  To implement this, the restriction options that don't have a Group attribute are presented in a list of radio buttons.  When a radio button is selected, a list of check boxes are presented from the options that have a Group attribute set to the same name as the selection non-group option.
