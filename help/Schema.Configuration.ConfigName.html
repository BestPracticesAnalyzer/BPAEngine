The ConfigName Attribute

This is a purely informational field that identifies the configuration.  It will likely have the name of the application that will process it and/or the product the configuration is designed to verify.  Although in most cases there will be only one configuration file per product, several may actually exist and each should be given a unique name so they can be distinguished.

This attribute is copied from the configuration to the output data file so it can be determined what configuration generated the data (this name along with the ConfigVersion should be unique).

Example Configuration Fragment

<Configuration
   ConfigVersion="1.6.7.3"
   ConfigName="ExBPA Master Input File"
   DownloadURL="http://www.microsoft.com/exchange/code/ExBPA"
   ArticleURL="http://go.microsoft.com/fwlink/?linkid=30939&amp;l={0}&amp;v=ExBPA.{1}&amp;id={2}"
   ChangeDescription="The changes made in this version are minor modifications to the configuration metadata."
>

Parent Element

Attribute Description
Configuration This element contains some general meta-information about the input configuration or the output data.
Run This output element contains general information about a particular run.

Possible Values

Value Description
<any> This can be any text string.
