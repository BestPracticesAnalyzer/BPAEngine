The Reference Element

This element refers to a Type definition.  The elements nested under the Type will be copied to the location identified by the Reference.  This is useful for defining reusable entities such as servers, files, disks, etc.  It is also useful for creating standard definitions that can be used by multiple applications.  A set of Common Models is provided as part of the SDK. These standard definitions will also come into play in the future when BPA configurations need to be converted to SDM configurations (the Systems Definition Model is a strategic effort being driven out of System Center that is being designed to handle best practice definitions and much more).  See the BPA to SDM Conversion Document for details. 

Note that Type, Reference, and Customize elements are only processed when the Type Configuration Preprocessor is configured.  See the built-in Configuration Preprocessor extensions for details.

Example Configuration Fragment

In the configuration file:

<Reference
   Type="File"
   File="BPA.CommonModels.xml"
   Name="NOTEPAD.EXE file"
>
   <Substitution Name="SRVNAME"></Substitution>
   <Substitution Name="FILENAME">NOTEPAD.EXE</Substitution>
</Reference>

Parent Elements

Element Description
root This element can be just below the XML root node.
Customize This element can be used to alter specific portions of referenced types.
Object On input, Object elements can only be nested under another Object element, or at the root of the document.
Setting This element identifies a property to be retrieved from an Object.
Type This element defines a reusable configuration fragment.

Child Elements

Element Description
Customize This element can be used to alter specific portions of referenced types.
Substitution This element contains a parameter value that can be references elsewhere in the configuration.

Attributes

Attribute Description
CustomId This is an identifier that can be referenced by a Customize element to override its properties.  This is only meaningful if the Reference is nested under a Type.  This defaults to the Name of the Reference.
File This identifies the file to look for the Type in.  This defaults to the current file.  If this is not the current file, then this name must also be included in the ReferencedFiles attribute so updates can be detected.
Name This provides information on the name of the Reference.  This name will be appended to the Rule Names and CustomIds of all elements in that Type.
SDMElement This provides a hint to be used during conversion of a BPA Object element to an SDM element.
SDMMultiplicity This provides a hint to be used during conversion of a BPA Object element to an SDM element.
SDMName This provides a hint to be used during conversion of a BPA Object element to an SDM element.
Type This is the name of the Type this Reference refers to.
