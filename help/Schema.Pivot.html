The Pivot Element

This element allow indirect references between rules, effectively acting like a pivot through which a rule can see another rule in a different part of the file. Pivoted references are made by prefixing the rule, e.g., Query="$pivot.rule".

Example Configuration Fragment

In the configuration file:

<Pivot
   Name="ControlServer"
   Key="../@Name"
   Remote="Server"
   KeyRef="$szBaselineControl[../../Value=$szExchangeServerNameCaps]"
/>

In an output data file:

<Pivot
   Name="ControlServer"
   Key="../@Name"
   Remote="Server"
   KeyRef="$szBaselineControl[../../Value=$szExchangeServerNameCaps]" 
/>

Parent Elements

Element Description
Object On input, Pivots are nested under Object elements.
Instance On output, Pivots are nested under Instance elements.

Child Elements

Element Description
None This element has no children.

Attributes

Attribute Description
AlwaysEvaluate If this is set, this pivot will be evaluated even if rules it refers to are not run.  Otherwise such pivots are skipped.
Key This is an XPath statement that should have a unique value for each instance of a given pivot.
KeyRef This is an XPath expression whose value should match the value of the Key of an instance of the remote pivot.
KeyRefValue On output, the result of the evaluation of the KeyRef attribute. 
KeyValue On output, the result of the evaluation of the Key attribute. 
Name This is a unique identifier of the pivot in the input file, composed of only letters and digits.
Remote This is used by indirect pivots to specify the name of the remote pivot. Any references using an indirect pivot will actually refer to a rule under the pivot whose Name matches this value, and whose KeyValue matches this pivot's KeyRefValue.
<restriction> This is a custom-defined restriction attribute.
