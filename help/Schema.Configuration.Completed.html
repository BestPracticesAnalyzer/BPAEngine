Best Practices Analyzer Core Library
The Completed Attribute

This attribute returns the completion status of a particular run. 

Example Configuration Fragment

 <Run
   ConfigName="ExBPA Master Input File"
   ConfigVersion="1.5.6.1"
   AppVersion="1.0.7408.1"
   StartTime="09/30/2004 23:20:39"
   Timeout="240"
   Threads="25"
   Operations="collect,analyze"
   DC="CONTOPGC01"
   EndTime="09/30/2004 23:25:48"
   Completed="OK"
   ObjectsTracked="9"
>

Parent Element

Attribute Description
Configuration This element contains some general meta-information about the input configuration or the output data.
Run This output element contains general information about a particular run.

Possible Values

Value Description
"OK" This indicates the run completed without a problem.  It does not indicate no issues were found, just that the run itself was successful.
"Aborted" This indicates a run was aborted by the calling process, which likely means it was aborted by a user.
"Failed" This means an internal failure occured that prevented the run from completing.  Since the BPA engine is designed to keep on running no matter what, this is likely to be a bug.
<not present> This means pretty much the same thing as Failed, except it is likely the result of the process being hard stopped or throwing an unhandled exception.
