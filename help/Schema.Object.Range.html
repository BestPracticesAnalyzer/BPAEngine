The Range Attribute

This indicates the maximum number of instances to include in the output.  If the Object resolves to more instances than this value, those excessive instances will be discarded.  This can be used in conjunction with the Sort attribute to only process a definable set of instances.  For example, if WMI is used to get event logs, Sort can order the logs by time written, and Range can limit it to just the last few event logs written.

Example Configuration Fragment

<Object
   Type="WMI"
   Key3="Win32_NTLogEvent"
   Key4="Logfile='Application' and EventType=1"
   Range="5"
>

Parent Element

Attribute Description
Object Range attributes are specified on input Object elements.

Possible Values

Value Description
0 This is equivalent to not having a range at all.  All instances found will be included in the output.  This is the default value.
<any positive number> The will cause all but the first x instances to be discarded.
